# Eval: elements-styling-layout

Answer from the skill + Source evidence. Each item: **Q → expected answer → proof**.

---

### 1. Why does `div().overflow_y_scroll()` fail to compile?

**Answer:** `overflow_y_scroll` is a default method on `StatefulInteractiveElement`, which is implemented for `Stateful<E>` (and `Img`), not for bare `Div`. Call `.id(...)` first so the type becomes `Stateful<Div>`.

**Proof:** `Source/gpui/src/elements/div.rs:1429` (method), `:3752-3757` (impl for `Stateful<E>`), `:710-714` (`.id` returns `Stateful<Self>`). Working sample: `Source/gpui/examples/scrollable.rs` (`.id("vertical").overflow_scroll()`).

---

### 2. Does this fork support max height on elements?

**Answer:** Yes. There is no field named `max_height`; the field is `Style.max_size.height` (`Size<Length>`). The Styled API is `.max_h(...)` / ramp methods like `.max_h_full()`, generated by `gpui_macros` into assignments on `max_size.height`, mapped to taffy.

**Proof:** `Source/gpui/src/style.rs:234` (`max_size`); `Source/gpui_macros/src/styles.rs:899-903` (`max_h` prefix); `Source/gpui/src/taffy.rs:496` (`max_size.to_taffy`).

---

### 3. How do you programmatically scroll a plain `div` to the bottom after new content?

**Answer:** Keep a `ScrollHandle` on the view, call `.id(...).overflow_y_scroll().track_scroll(&handle)`, then `handle.scroll_to_bottom()` after content grows. The flag is applied on the next scroll clamp (`y = -scroll_max.y`), not as a continuous follow mode.

**Proof:** `ScrollHandle::scroll_to_bottom` at `div.rs:4063-4066`; flag consumption `div.rs:2244-2276`; `track_scroll` at `div.rs:1435-1438`. Continuous tail-follow for variable-height lists is `ListState::set_follow_mode(FollowMode::Tail)` (`list.rs:617-635`), not `ScrollHandle`.

### 4. `overflow_hidden` vs `overflow_y_scroll` — which trait?

**Answer:** `overflow_hidden` / `overflow_x_hidden` / `overflow_y_hidden` are on **`Styled`** (macro-generated). `overflow_scroll` / `overflow_x_scroll` / `overflow_y_scroll` are on **`StatefulInteractiveElement`**.

**Proof:** macro at `Source/gpui_macros/src/styles.rs:135-151`; scroll methods at `div.rs:1416-1432`; `Styled` expands overflow macros at `styled.rs:31`.

---

### 5. When do you pick `uniform_list` vs `list`?

**Answer:** `uniform_list` when every row has the **same height** (measures one item, lays out by arithmetic; range callback). `list` + `ListState` when heights **vary** (SumTree of heights; per-index render; must `splice`/`reset`/`remeasure` if off-screen heights change).

**Proof:** module docs `uniform_list.rs:1-5` and `list.rs:1-8`; constructors `uniform_list.rs:22-55`, `list.rs:24-34`. Examples: `uniform_list.rs`, `list_example.rs`, scale: `data_table.rs` (10k uniform rows).

---

### 6. How do you ellipsis-truncate a single-line label?

**Answer:** Give the element a definite width and use `.truncate()` (nowrap + overflow hidden + end ellipsis) or `.text_ellipsis()` with nowrap as needed. Multi-line clamp uses `.line_clamp(n)`.

**Proof:** `styled.rs:87-92` (`text_ellipsis`), `:137-141` (`truncate`), `:143-149` (`line_clamp`). Matrix: `examples/text_wrapper.rs`. Layout needs known width (`text.rs` truncate path).

---

### 7. Where does text selection live in this monorepo?

**Answer:** Not in core `gpui` text elements (`InteractiveText` is click/hover/tooltip only). Selection demo is under **gpui-component** (`examples/text_selection/`), which ChronOS does not depend on today.

**Proof:** `Source/gpui/src/elements/text.rs` (no selection API on Text/StyledText); `Source/gpui-component/examples/text_selection/`.

---

### 8. Why might `grep` fail to find `max_h` or other style helpers in `gpui/src`?

**Answer:** Many Styled helpers are generated by `gpui_macros::style_helpers!` and related procs; they do not appear as handwritten `fn max_h` in `gpui/src`. Look in `gpui_macros/src/styles.rs` (`BoxStylePrefix` entries) and the macro invocations on `Styled` in `styled.rs:26-34`.

**Proof:** `styled.rs:26` (`style_helpers!()`); `gpui_macros/src/styles.rs:891-903` (`max_w`/`max_h` prefixes).
