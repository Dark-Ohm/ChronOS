# ChronOS — Roadmap

Форвард-лук, одностраничный. Не дублирует детали — за рационализацией
решений см. `DECISIONS.log`, за канонической архитектурой — `ARCHITECTURE.md`,
за оперативным полем (кто чем занят прямо сейчас) — `HANDOFF.md`. Этот файл
отвечает на один вопрос: «куда мы идём и в каком порядке».

Обновлено: 2026-07-24 (вечер) — годовой план впервые зафиксирован ниже.

## Северная звезда (позиционирование, `DECISIONS.log` 2026-07-24)

ChronOS — не очередной Hyprland-rice конфиг поверх Quickshell/AGS (end-4/
dots-hyprland, ml4w, dank-material-shell). Это первого лица продукт на
GPUI — незанятая ниша (ни один популярный Hyprland-шелл не написан на
GPU-accelerated retained-mode тулките с нуля). Личный инструмент, но с
прицелом на community/публичный резонанс. Следствие: plugin-экосистема
должна реально мочь то, чего нет у конфиг-шеллов — иначе нет причины
мигрировать (см. "Plugin API v2" ниже, находка 2026-07-24).

## Годовой план (2026-07 → 2027-07)

Кварталы — приоритет и порядок, не жёсткие дедлайны (однопользовательский
проект, скорость = сколько миньонов в поле). Детали каждого пункта — в
секциях ниже по файлу или в `DECISIONS.log`/`HANDOFF.md`, здесь только
последовательность и обоснование порядка.

### Q1 (2026-08 — 2026-09) — Добить Shell-IDE каркас, вскрыть Plugin API v2

Всё, что уже в поле или прямо перед стартом — закрыть, прежде чем
открывать Editor (самый крупный и рискованный кусок).

1. **3 вкладки IDE-панели** (T113 Terminal, T114 ACP settings, T115
   Files) — уже розданы, приёмка по мере готовности отчётов.
2. **`desktop_terminal` — из спайка в продукт.** Пользователь: "виджет
   терминала на десктопе которым я не могу пользоваться" — это
   ПРЕДПОСЫЛКА для T113 (Terminal-таб хочет переиспользовать его PTY/
   VT100 начинку), значит чинить его надо ДО или ВМЕСТЕ с T113, не после.
   Живой баг-репорт от пользователя — что именно не работает — нужен
   первым шагом.
3. **MPRIS-карточка — тоже живой баг.** "мокап плеера который не
   заработает" — отдельная триаж-задача, не часть T113/114/115.
4. **Bar widget layout config** — спека готова, роздать план+T-задачу.
5. **Plugin API v2, фаза 1** (находка 2026-07-24: `capabilities.rs` —
   полная permission-система с тестами, но `mod_fs`/`mod_process`/
   `mod_net`/`mod_ipc` — буквально `lua.create_table()`, пустые заглушки;
   `manager.rs` даже не вызывает `has_capability()` — проверка прав
   существует, но ни к чему не подключена). Фаза 1: реализовать `fs`
   (read-only сначала) и `spawn` по-настоящему, подключить
   `has_capability()` в реальный вызов, render DSL — добавить `image`/
   `on_click` (сейчас только `text`/`row`/`column`). Без этого community
   физически не может написать ничего сложнее перекраски текста — это
   named gap, не абстрактная возможность (см. `DECISIONS.log`).

### Q2 (2026-10 — 2026-12) — Editor-таб, фундамент

Крупнейший и самый рискованный фронт всего года. Начинается только
после Q1, чтобы не распылять внимание на два тяжёлых фронта одновременно.

1. **Recon-skill по паттернам Zed `text`/`rope`/`sum_tree`**
   (`/home/neo/Projects/zed-main/crates/{text,rope,sum_tree}`) — GPL-3.0
   у `text`/`rope`, но `sum_tree` (Apache-2.0) уже форкнут у нас
   (`Source/gpui_sum_tree`). Read-only референс, 0 скопированных строк,
   по образцу `zed-message-editor`/`zed-thread-view`.
2. **Собственный крейт `chronos-editor`** (пользователь уже завёл папку
   под него) — rope-эквивалент поверх нашего `gpui_sum_tree`, buffer с
   line/offset/point конверсиями, базовые edit-операции. Отдельное
   репо/крейт, интегрируется в Editor-таб как обычная Rust-зависимость
   (`Entity`/`Render`, как `crates/hotview`), НЕ как Luau-плагин — плагин-
   слой физически не тянет rope-буфер с рендер-критичной производительностью.
3. **v1 функциональность:** открыть/сохранить файл, plain-text (без
   подсветки), моноширинный грид-рендер, курсор/выделение, базовые
   кейбинды. Синтаксическая подсветка и LSP — явно НЕ в этом квартале
   (см. Q3).
4. **Plugin API v2, фаза 2** (параллельно, другой минион): `network`
   capability, canvas/raw-paint escape hatch в render DSL — примеры
   плагинов, показывающие потолок выше cava.

### Q3 (2027-01 — 2027-03) — Editor-таб зрелость + оставшиеся вкладки

1. **Синтаксическая подсветка** — tree-sitter (сам крейт MIT, грамматики
   отдельно лицензированы по каждой) поверх `chronos-editor`, свой query-
   wiring, не порт `language`-крейта Zed (GPL).
2. **LSP-вкладка + базовое LSP в Editor** — scope узко: diagnostics/hover/
   go-to-def первой итерацией, полный LSP-клиент это уже отдельная веха.
3. **MCP settings / API-providers вкладки** — каждой нужен новый
   `crates/services/src/{mcp,api_providers}/` с нуля (в дереве сегодня
   нет вообще).
4. **Hyprland binds вкладка** — новый `crates/services/src/hyprland_binds/`
   (парсер + watcher `hyprland.lua`).

### Q4 (2027-04 — 2027-06) — Экосистема, полироль, публичная готовность

1. **Plugin API v2, фаза 3** — showcase-плагины (реальный виджет
   сложнее cava, чтобы было что показать community), документация API.
2. **Светлая тема** — порт в `crates/ui/src/theme/schemes.rs::light_scheme()`
   (айдентика решена ещё 2026-07-19, порт не начат).
3. **Gradient borders** — порт Kael-вырез из `Source/` (фундамент уже
   принят).
4. **`ccf-gpui-widgets` вендоринг** (текстовый ввод с курсором для
   композера левой панели — C-2 gap из T109, живой, не абстрактный).
5. **README/публичная документация — pass на чужие глаза.** Учитывая
   Северную звезду (публичный резонанс, не приватный дотфайл-репо) —
   этот пункт целенаправленно ждёт, пока Editor-таб и Plugin API v2 дадут
   что показать, не делается раньше времени.

### Не запланировано намеренно (отклонено или вне года)

- **WASM как второй plugin-рантайм** — отклонено 2026-07-24, не решает
  реальное узкое место (host API, не исполнитель скрипта), см.
  `DECISIONS.log`.
- **Chronos-lm (сиблинг-проект)** — отдельный трек, отдельная сессия, не
  смешивать с этим roadmap (`split-sessions-per-project` в памяти
  архитектора).
- **Полный порт Zed `editor`/`multi_buffer`/`language`** — GPL-3.0,
  архитектурно "половина Zed" — не рассматривается, только study-паттерн
  для собственного `chronos-editor`.

## Текстовый ввод — `ccf-gpui-widgets` (разведано 2026-07-20)

Самая продуктово значимая находка сегодняшнего дня — закрывает
named gap, а не абстрактную возможность. `DECISIONS.log` 2026-07-16
отказался от текстового ввода Kael как overkill («2987+243 LOC +
отсутствующий undo_manager… launcher-ввод добить руками») и оставил
заглушку. Boковая agent-панель с чатом (см. выше) без многострочного
текстового поля с курсором/историей/фокус-навигацией не строится.

`ccf-gpui-widgets` (18000 строк, v0.1.0) даёт ровно это:
`text_input.rs`/`cursor_blink.rs`/`editing_core.rs`/`focus_navigation.rs`
+ россыпь готовых виджетов (`file_picker`, `dropdown`, `slider`,
`toggle_switch`, `confirmation_dialog` и др. — не всё нужно, но список
широкий). `gpui = "0.2.2"` голый (версия совпадает с нашим форком),
`smol` внутри — паттерн executor'а не проверен (тот же класс вопроса,
что у `gpui-animation`).

**Трение — их собственная плоская `Theme` (`u32`-поля:
`bg_primary`/`text_primary`/…), не `gpui-component`.** Дешевле, чем
казалось: структура семантически близка нашей, виджеты принимают
`custom_theme: Option<Theme>` явно — маппинг `chronos_ui::Theme` → их
`Theme` это одна функция на десяток полей, не архитектурная работа.

**Разведано компиляцией 2026-07-21 (Hermes) — вердикт: ВЕНДОР ОБЯЗАТЕЛЕН,
ОТЛОЖЕН.** НЕ компилируется против форка: 58 ошибок, 5 классов форк-дельт
(`FocusHandle::focus(+cx)` ×43, `update_entity→R` ×3, `Corner` удалён,
`Bounds::from_corner_and_size` удалён, `ScrollHandle::max_offset` сменил
тип). Animation-паттерн, не rsx — `[patch]`-зависимостью не подключить.
Ввод хирургически извлекаем (`text_input+editing_core+cursor_blink+
focus_navigation`, 4 файла), но дельты чинить ровно в них. Тема-маппинг
нетривиален: их 52 плоских `u32`-поля vs наши `Hsla`-группы → адаптер
`Hsla→u32` ~40 полей + коллизия `gpui::Global`. **Отложено:** named gap
реален, но не на критическом пути правой панели; вернёмся отдельной
задачей к agent-панели. Отчёт: `report-log/hermes-report-widgets-recon.md`.

**Апдейт 2026-07-23 — gap теперь на критическом пути.** T109 (левая
agent-панель, Agent Thread canvas) сдана; живой смок пользователем сразу
поймал ровно то, что здесь предсказано: "нет моргающей палочки когда я
ввожу" — самопальный `text_input` в `composer.rs` не рисует курсор,
выделение, wrap-навигацию (гарантированный fallback из брифа T109 C-2,
`gpui-component` заблокирован конфликтом версий gpui — форк
`zed-industries/zed` git-gpui vs наш `gpui-ce`, `E0432` на
`AsyncApp/Result/SharedString`). Вендоринг `ccf-gpui-widgets` — тот же
5-класс форк-дельт (`FocusHandle::focus(+cx)`, `update_entity→R`, `Corner`
удалён и т.д.), что и в разведке, но теперь оправдан: реальный пользователь
уже упёрся в реальный gap на реальной панели, не абстрактная возможность.
Следующий шаг — отдельная задача на хирургический вендоринг
(`text_input+editing_core+cursor_blink+focus_navigation`, 4 файла + тема-
адаптер), не блокирует остальной T109-канвас.

## Кандидаты на переходы/анимацию — `gpui-animation` (разведано 2026-07-20)

Не привязано к панелям, применимо уже сегодня. `gpui = "0.2.2"` без

.detach()` — сама библиотека соблюдает правило, которое мы весь день
ловили как баг у своих виджетов), `smol::Timer` — просто future внутри
таски, третьего рантайма не заводит. Даёт декларативный слой ПОВЕРХ
уже вшитых в форк `EasingCurve` (`gpui/src/easing.rs`, находка DeepSeek
№2): `.transition_on_hover(duration, curve, |state, style| ...)`,
умная интерполяция HSLA по кратчайшему пути, откат при прерывании одной
анимации другой. Прямой кандидат применения — hover шести виджетов
правого кластера бара (Mimo №11, сейчас мгновенный щелчок цвета через
голый `.hover(|s| s.bg(...))`) стал бы плавным затуханием без ручной
анимационной логики. Пометка: README называет себя «early development,
API subject to change» — готовиться к breaking changes при апдейте.

**Разведано компиляцией 2026-07-20 (Grok №18) — статус НИЖЕ, чем у
gpui-form/gpui-rsx.** НЕ компилируется out-of-the-box: 3 API-дельты
форка против crates.io `0.2.2` (`AsyncApp::update` non-fallible,
`BoxShadow.inset` — лишнее поле, `Style.text` `#[refineable]`).
После 3 хирургических правок их исходников (~15 строк) — компилируется
и живой hover-переход подтверждён grim (тёмный → промежуточный синий →
целевой голубой, не snap). **Правило для будущих разведок:** совпадение
семвера (`0.2.2` у обеих сторон) НЕ значит совместимость ABI, когда одна
сторона — git/path-форк, а другая — crates.io. Взять эту библиотеку =
взять на сопровождение форк её кода (patch+vendored tree), не просто
добавить строку в Cargo.toml.
