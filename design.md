# ChronOS — design brief for Claude Design

## Что это

ChronOS — desktop shell для Hyprland/Niri: bar, dock, launcher,
notifications, OSD, контекстные попапы. Рендерится нативным GPUI
(Rust), НЕ веб-стеком. Дизайн-система здесь — референс-борд, не
источник кода: макеты в HTML/CSS, компонент переносится в GPUI
руками отдельным агентом. Задача Claude Design — визуальные
мокапы конкретных попапов ниже, в точной палитре проекта, без
отсебятины по цвету/spacing.

## Палитра (источник истины — `crates/ui/src/theme/schemes.rs`, Base16)

Два готовых статических варианта в коде — `default_scheme()` (тёмная,
живой дефолт, на ней сейчас бар/попапы/лендинг) и `light_scheme()`
(светлая, тоже реальный код, не заглушка, просто сейчас не включена
по умолчанию). **Мокапь оба варианта каждого компонента** —
компоненты должны быть теми-agnostic (ссылаться на роли
`bg.primary`/`text.muted`/`accent.primary`, а не на конкретный hex),
чтобы переключение схемы ничего не ломало по layout/contrast.

Важный контекст на будущее (не реализовано сейчас, не мокапь под
это отдельно, но держи в голове почему роли важны больше hex-ов):
финальное направление — тема должна подстраиваться под цвет обоев
(wallust/matugen-подобная динамическая палитра), а не быть жёстко
зашитой парой dark/light. В дереве такой экстракции цвета из
wallpaper пока НЕТ (проверено — ни wallust, ни pywal, ни matugen не
подключены), это отдельная будущая задача, не часть этого брифа.
Практический вывод для мокапов: не завязывайся на конкретные hex
жёстко в композиции (например, не рисуй элемент, который выглядит
только с этим синим акцентом) — используй роли как ниже, они не
изменятся, даже когда сами цвета станут динамическими.

### Тёмная (дефолт)

```
bg.primary        #1e1e2e   основной фон окна/бара
bg.secondary      #25253b   divider / secondary surface
bg.tertiary       #181825   самый тёмный слой (overlay/nav)
bg.elevated       #313244   карточки/попапы поверх bg.primary

text.primary      #cdd6f4   основной текст
text.secondary    #a6adc8   вторичный текст
text.muted        #6c7086   приглушённый (лейблы, таймстемпы)
text.disabled     #45475a   выключенное состояние

accent.primary    #007acc   акцент/кнопки/фокус-бордер
accent.hover      #cba6f7   hover-состояние акцента

status.error      #f87171
status.warning    #fbbf24
status.success    #4ade80
status.info       #60a5fa
```

### Светлая (`light_scheme()`, инверсия bg/text того же дефолта)

```
bg.primary        #eff1f5
bg.secondary      #e6e9ef
bg.tertiary       #f7f8fb
bg.elevated       #dce0e8

text.primary      #4c4f69
text.secondary    #5c5f77
text.muted        #9ca0b0
text.disabled     #bcc0cc

border.default    #ccd0da
border.subtle     #e6e9ef
interactive.hover  #bcc0cc
interactive.active #9ca0b0
```
accent/status в светлой теме НЕ переопределены в коде — те же hex,
что в тёмной (`accent.primary #007acc` и т.д.); если на глаз это
не будет читаться на светлом фоне — отметь как находку в отчёте
дизайна, не переопределяй сам, это код-задача.

Шрифты: моно (`JetBrains Mono`) для чисел/статусов/технических
элементов (как в баре — часы, проценты, workspace-номера), sans
(`Inter`) для текстовых лейблов. Радиусы — небольшие, 6-10px
(`theme.radius`/`theme.radius_lg`), не круглые пилюли. Общий
характер — тёмный, компактный, техничный, без градиентов и теней
«под iOS» (это не macOS-виджет, это компоузитор-нативный shell).

## Формат окон (важно для мокапов — это не веб-страницы)

Всё, что ниже — **layer-shell попапы поверх бара**, не полноэкранные
страницы. Компактные (300-400px шириной), привязаны углом к иконке
в баре (обычно `TOP|RIGHT`), с закруглёнными углами и `bg.elevated`
подложкой. Мокапь их именно как плавающие карточки на тёмном фоне
рабочего стола, не как full-page UI.

## Компоненты, которым реально нужен визуал (в приоритете)

### 1. Volume/mic popup (в работе, Grok №12)
Два блока — Speakers и Microphone. У каждого: название, fill-bar
(горизонтальная полоса-индикатор уровня, залитая `accent.primary`
на `bg.tertiary`-треке — БЕЗ драга мышью, чисто визуальный уровень),
процент числом справа (моно-шрифт), три мелких контрола под баром:
`−5%` / mute-иконка (динамик/микрофон, зачёркнутый в состоянии
mute) / `+5%`. Два таких блока друг под другом, разделённые тонкой
`bg.secondary`-линией.

### 2. Updates popup (принят, но выглядит «дёшево» — сам автор
   это признал, повод для рестайла)
Уже живой компонент: заголовок «Updates (N)» + крестик закрытия,
список пакетов (имя [+ `(AUR)` тег] слева, `старая → новая версия`
справа приглушённым текстом), кнопка «Upgrade all» на всю ширину
внизу (`accent.primary` фон, текст `bg.primary`, hover →
`accent.hover`). Референс структуры — `crates/app/src/updates_popup/
view.rs`, если нужно свериться с реальным деревом элементов.

### 3. Notifications card
Уведомление: заголовок (bold, `text.primary`) + body (обычный,
`text.secondary`, может быть несколько строк) + опциональные
action-кнопки снизу. Несколько карточек стопкой, разделённые
тонкими линиями `bg.secondary`. Urgency (`status.error`/`.warning`/
`.info`) — тонкая цветная полоса слева от карточки, не заливка
всей карточки.

### 4. Tray context menu (DBusMenu popup)
Компактное вертикальное меню — пункты с опциональными иконками,
разделители между группами, инлайн-submenu (вложенные пункты со
сдвигом). Тёмный `bg.elevated`, hover-строка — `bg.tertiary`.

### 5. Dock (закреплённая панель запуска)
Горизонтальный ряд квадратных иконок на `bg.primary`/`bg.tertiary`
подложке снизу экрана, с небольшим отступом между иконками и
подсветкой hover (`bg.secondary` за иконкой). Fallback без реальной
иконки — первая буква имени приложения в кружке.

### 6. System popup (яркость + режим питания + gaming mode)
Заменяет собой battery-иконку в баре на десктопах без физической
батареи (там иконка сейчас пустая — кликать не по чему). Popup
структурно как Volume — узкая карточка, header «System» + крестик,
три блока друг под другом, разделённые тонкой `bg.secondary`-линией:

- **Brightness** — тот же fill-bar паттерн, что Volume (не
  интерактивный визуальный уровень + `−5%`/`+5%`-степперы), иконка
  солнца слева, процент справа моно-шрифтом. Нет toggle/mute для
  этого блока (яркость не мьютится).
- **Power profile** — трёхпозиционный сегмент-переключатель
  (Quiet / Balanced / Performance) в один ряд, активный сегмент
  залит `accent.primary`, текст `bg.primary`; неактивные —
  прозрачные с `border.subtle`, текст `text.muted`. Не выпадающий
  список — все три опции видны сразу, это частое переключение.
- **Gaming mode** — один toggle-переключатель (свитч, не кнопка):
  подпись «Gaming mode» слева, свитч справа (OFF: серый трек с
  кружком слева, ON: `accent.primary`-трек с кружком справа —
  стандартный iOS-style toggle, radius=full). Под свитчем — мелкая
  приглушённая строка `text.muted`, перечисляющая, что реально
  включает режим (так пользователь видит эффект, не гадает):
  «Performance profile · No animations · Do Not Disturb · Hide
  bar/dock · VSync forced» — можно сократить/перенести на 2 строки,
  главное чтобы это было видно, не спрятано за tooltip.

Иконка вызова в баре (замена battery-слота) — не батарея, что-то
нейтральное типа шестерёнки/слайдеров (не рисуй разряд батареи, тут
её физически нет на десктопе).

## Что НЕ нужно

Не предлагай смену палитры, шрифтов, общего стиля («сделаем
светлее/веселее/glassmorphism») — это уже согласованная система,
задача — довести конкретные попапы до уровня остального, не
переизобрести язык. Не рисуй полноэкранные web-подобные layouts —
все компоненты выше маленькие, угловые, поверх десктопа.
